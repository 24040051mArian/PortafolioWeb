1. Paradigma del desarrollo de aplicaciones orientadas a servicios

El desarrollo de aplicaciones orientadas a servicios es un paradigma que se basa en la creación de servicios independientes, reutilizables y comunicados entre sí a través de la red. Estos servicios permiten que distintas aplicaciones interactúen sin importar el lenguaje o la plataforma en la que fueron desarrolladas.

Servicios que se ofrecen en la nube

Los servicios en la nube permiten acceder a recursos tecnológicos a través de Internet sin necesidad de instalarlos localmente. Los principales tipos son:

  • IaaS (Infrastructure as a Service): Proporciona infraestructura como servidores, redes y almacenamiento.
    Ejemplo: AWS EC2, Azure Virtual Machines.
  • PaaS (Platform as a Service): Ofrece plataformas para desarrollar y desplegar aplicaciones.
    Ejemplo: Google App Engine, Heroku.
  • SaaS (Software as a Service): Proporciona aplicaciones listas para usar desde el navegador.
    Ejemplo: Gmail, Google Drive, Microsoft 365.

Características de las aplicaciones orientadas a servicios

  • Modularidad: Cada servicio cumple una función específica.
  • Reutilización: Un servicio puede ser usado por distintas aplicaciones.
  • Interoperabilidad: Funcionan entre diferentes sistemas y lenguajes.
  • Escalabilidad: Pueden crecer según la demanda.
  • Bajo acoplamiento: Los servicios son independientes entre sí.

Aplicaciones web híbridas (Mashup)

Las aplicaciones web híbridas combinan información o funcionalidades de varios servicios o APIs en una sola aplicación.

  • Integran datos de diferentes fuentes.
  • Mejoran la experiencia del usuario.
  • Son dinámicas e interactivas.
  • Utilizan APIs públicas o privadas.

Ejemplo: Una app que muestra mapas de Google Maps con datos del clima obtenidos de otra API.

2. Arquitectura Orientada a Servicios (SOA)

¿Qué es la Arquitectura Orientada a Servicios (SOA)?

La Arquitectura Orientada a Servicios (SOA) es un modelo de diseño de software donde las funcionalidades del sistema se organizan como servicios independientes, accesibles a través de la red y reutilizables por distintas aplicaciones. Su objetivo principal es facilitar la integración, escalabilidad y mantenimiento de los sistemas.

Principios de diseño de SOA

  • Autonomía: Cada servicio controla su propia lógica.
  • Reutilización: Los servicios pueden usarse en distintos sistemas.
  • Bajo acoplamiento: Cambios en un servicio no afectan a otros.
  • Interoperabilidad: Comunicación mediante estándares.
  • Descubrimiento: Los servicios pueden ser localizados y utilizados fácilmente.

Estándares relacionados con los servicios

  • XML (eXtensible Markup Language): Lenguaje de marcado para estructurar y transportar datos de forma legible entre sistemas.
  • SOAP (Simple Object Access Protocol): Protocolo que permite el intercambio de mensajes estructurados en XML entre aplicaciones.
  • WSDL (Web Services Description Language): Lenguaje que describe cómo funciona un servicio web, qué operaciones ofrece y cómo acceder a él.
  • UDDI (Universal Description, Discovery and Integration): Registro que permite publicar y encontrar servicios web disponibles.
  • REST (Representational State Transfer): Estilo de arquitectura que utiliza HTTP para acceder a recursos mediante métodos como GET, POST, PUT y DELETE.
